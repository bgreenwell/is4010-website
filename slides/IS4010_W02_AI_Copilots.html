AI tools for programming

Welcome to the AI revolution

  • Today we’ll explore the landscape of AI programming tools
  • You’ll learn to work with AI as your pair programming partner
  • We’ll compare the major players and find the right tool for each job
  • By the end, you’ll understand how to “talk” to AI about code

Start with enthusiasm - this is genuinely exciting technology that will transform how students approach programming. Emphasize that we’re not replacing programming skills, but augmenting them.

Demo hook: Have three browser tabs open (Gemini, ChatGPT, Claude) and be ready to show the same prompt to all three for the live comparison.

Course motto:

Augment, don’t automate!

  • AI is your co-pilot, not your autopilot
  • Think of AI as a brilliant but sometimes eccentric junior developer
  • AI provides suggestions → You review, approve, and own the code
  • Your job: Direct, verify, and integrate AI assistance
  • Remember: The pilot is always responsible for the flight

This is the foundational philosophy for the entire course. Spend time here - students need to understand that they remain in control and responsible for all code they submit.

Use the pilot/co-pilot analogy throughout the course. It’s memorable and accurate.

What are AI programming tools?

  • Code generation: Turn comments and prompts into working code
  • Code explanation: Break down complex functions into plain English
  • Debugging assistance: Help identify and fix errors
  • Refactoring support: Improve code structure and style
  • Learning accelerator: Get instant feedback and alternative approaches

Under the hood: LLMs 🧠

  • Large Language Models (LLMs): AI systems trained on massive amounts of text from the internet
  • Pattern recognition experts: They’ve learned patterns in human language and code
  • Sophisticated autocomplete: Like your phone’s predictive text, but incredibly advanced
  • Not databases: They don’t “know” facts—they predict what text should come next
  • Surprisingly good at code: Programming languages follow patterns, just like natural language

This is crucial foundational knowledge. Students often think AI “knows” things, when really it’s predicting based on patterns.

Key analogies to use: - Autocomplete on steroids: Everyone understands phone predictive text - Pattern matching: Like learning to recognize handwriting, but for language/code patterns - Very sophisticated parrot: Can produce remarkably coherent text without true “understanding”

Important points: - LLMs were trained on billions of lines of code from GitHub and other sources - They learned that certain patterns appear together (if you write a function definition, a return statement usually follows) - This is why they’re good at code completion but can make subtle logical errors - They work the same way for Python, Rust, English, or any other language—it’s all just patterns in text

Set expectations: These tools are incredibly useful but require human judgment and verification.

Four main categories:

  • Conversational AI models: General-purpose models (Gemini, ChatGPT, Claude)
  • Specialized code assistants: Purpose-built for coding (GitHub Copilot, Claude Code)
  • Command-line tools: Terminal-based AI access (Gemini CLI, Copilot CLI)
  • Code analysis tools: Project-wide review and feedback (Google Jules)

Frame these as the five core use cases students will encounter most frequently. Every AI interaction should fit into one of these categories.

Ask students: “Which of these sounds most useful to you right now?” Get a sense of where they are in their programming journey.

Conversational coding 💬

The CPTF framework for great prompts:

  1. Context: “Here is my Python function…”
  2. Persona: “Act as a senior software developer reviewing my code.”
  3. Task: “Find any potential bugs and suggest improvements.”
  4. Format: “Provide your answer as a numbered list.”

This is crucial - the quality of AI responses depends entirely on prompt quality. Students often start with vague prompts like “fix my code” and get frustrated with the results.

Have a few examples ready: - Bad: “Make this better” - Good: “Act as a Python expert. Here’s my function that calculates fibonacci numbers. Explain how it works and suggest any performance improvements. Format your response with explanations and code examples.”

Five essential prompt patterns

1. Explain this code

"Explain this Python dictionary comprehension step by step:

{word: len(word) for word in text.split() if len(word) > 3}"

2. Generate boilerplate

"Write a Numpy-style docstring for this function that calculates the area of a circle given its radius."

3. Refactor for clarity

"Convert this for loop into a more Pythonic equivalent using list comprehension or built-in functions."

Walk through each pattern with a concrete example. These are the patterns students will use 80% of the time.

Emphasize that good prompts are specific and provide context. The AI needs to understand what you’re trying to accomplish.

More essential patterns

4. Debug assistance

"I'm getting a 'TypeError: unsupported operand type(s)' on line 15. Here's my code... What's the likely cause and how do I fix it?"

5. Architecture advice

"I need to store student information (name, ID, grades). Should I use a list of dictionaries or create a Student class? Explain the pros and cons of each approach."

Pattern 4 is incredibly valuable - students often get cryptic error messages and spend hours debugging what an AI can spot in seconds.

Pattern 5 shows how AI can help with design decisions, not just code generation. This is where the “senior developer” persona really shines.

A quick note on docstrings 📝

  • Before our lab, we need to introduce a key concept: the docstring
  • A docstring is a string literal, enclosed in triple quotes ("""), that occurs as the first statement in a module, function, class, or method definition
  • Its purpose is to explain what the code does
  • We are learning about them now because they are the primary way you will communicate your intent to AI assistants
  • A good docstring leads to good AI-generated code

This bridges from the AI discussion to the practical lab work. Students need to understand docstrings before Lab 02 Part 1.

Emphasize the connection: clear human communication (docstrings) leads to better AI assistance. This reinforces that AI tools respond to human intent and clarity.

Our docstring style: NumPy

  • There are several popular styles for writing docstrings. In this course, we will use the NumPy style
  • It is clean, readable, and works very well with automated documentation tools
  • See next slide for an example

An example of Numpy-style docstrings:

def add(num1, num2):
    """
    Add two numbers together.

    Parameters
    ----------
    num1 : int
        The first number.
    num2 : int
        The second number.

    Returns
    -------
    int
        The sum of the two numbers.
    """
    return num1 + num2

Show the NumPy docstring format that will be used throughout the course. This is important for Lab 02 where students will be working with docstrings extensively.

Point out: - The brief description first - Parameters section with types and descriptions
- Returns section with type and description - Clear, consistent formatting

This style works particularly well with AI tools because it provides structured, detailed context about function intent.

Make sure students have VS Code open and ready for the GitHub Copilot demonstration.

What’s GitHub Copilot?

  • Purpose-built for coding and integrates directly into VS Code!
  • Trained specifically on code repositories from GitHub
  • Provides two main modes of assistance:
    • Inline suggestions: “Ghost text” appears as you type
    • Copilot Chat: Conversational interface within your editor
  • Context-aware: Sees your current file, open files, and project structure

This is the primary specialized code assistant for the course. Make sure students understand that Copilot is fundamentally different from the conversational AI models - it’s purpose-built for code generation and editing.

Key point: Context awareness and integration. Copilot can see variable names, function signatures, imports, and other files in your project. This makes its suggestions much more relevant than general-purpose AI models working with isolated code snippets.

The magic of inline suggestions ✨

How it works:

  • Write a descriptive comment about what you want to accomplish
  • Start coding…
  • Copilot appears as gray “ghost text”
  • Press Tab to accept, Esc to reject
  • Use Alt + ] (Windows) or Option + ] (Mac) to cycle through alternatives

The magic of inline suggestions ✨

Best practices:

  • Clear comments are key:

    # Calculate the factorial of a positive integer

  • Descriptive function names:

    calculate_factorial not calc_fact

  • Type hints help:

    def factorial(n: int) -> int:

This is the “classic” Copilot experience that most people think of. Prepare a live demo:

  1. Open a new Python file
  2. Write a comment: “# A function that calculates the factorial of a number”
  3. Start typing: “def factorial(n: int) -> int:”
  4. Let Copilot complete the function
  5. Show how to cycle through alternatives

Emphasize that the quality of suggestions depends heavily on the quality of your comments and function names.

Command-line AI assistants 💻

Why use terminal-based AI?

  • Stay in your workflow: No need to switch between terminal and browser
  • Quick queries: Fast answers without leaving the command line
  • Scriptable: Can be integrated into automated workflows
  • Code context: Easy to pipe file contents directly to AI
  • Privacy options: Some tools offer better data handling

Set up the value proposition for command-line tools. These are for “power users” who want to integrate AI into their terminal workflow.

Students may ask: “Why not just use the browser?” Answer: When you’re deep in terminal work (git operations, file manipulation, debugging), switching to a browser breaks your flow.

Gemini CLI

(Free) AI in your terminal 🖥️

Common usage patterns:

# Start an interactive gemini session
gemini

# Quick code questions
gemini "How do I install [pytest](https://pytest.org/) in [Python](https://www.python.org/)?"

# Code generation
gemini "Write a [Rust](https://www.rust-lang.org/) function to read a CSV file"

# Debugging help
gemini "Explain this error: ImportError: No module named 'requests'"

Prepare a live demo of Gemini CLI. Have it installed and configured beforehand.

Show practical examples: 1. Quick package installation help 2. Language translation (Python to Rust) 3. Error message explanation 4. Code review of a file: cat script.py | gemini "Review this code for bugs"

GitHub Copilot CLI

(Free tier) Natural language terminal ⌨️

Common usage patterns:

# Ask how to do something in the terminal
gh copilot suggest "list all python files larger than 1MB"

# Explain a complex command
gh copilot explain "find . -name '*.py' -size +1M"

# Execute suggestions directly
# (Follow the prompts to execute or copy)

GitHub Copilot CLI is part of the gh (GitHub CLI) tool. It’s excellent for when you know what you want to do in the terminal but forget the exact flags or syntax.

Key feature: It doesn’t just give you code; it gives you terminal commands and explains what they do before you run them.

Your professional AI workflow 🔄

Daily development:

  1. Start with comments: Describe your intent clearly
  2. Use Copilot inline: Accept, modify, or reject suggestions
  3. Chat for complex problems: Switch to conversational help when stuck
  4. Command-line for quick help: Terminal-based queries for efficiency

Your professional AI workflow 🔄

Code review and submission:

  1. Jules analysis: Upload complete projects for comprehensive review
  2. AI-assisted debugging: Use chat to identify and fix issues
  3. Documentation: Generate docstrings and comments with AI help
  4. Testing: Auto-generate test cases with /tests command

This gives students a complete workflow they can start using immediately.

Emphasize the progression: start with simple inline suggestions, graduate to conversational help for complex problems, and use specialized tools for specific tasks like code review.

The key insight: different tools for different phases of development. You wouldn’t use the same tool for initial brainstorming as you would for final code review.

Remember: You are the pilot ✈️

  • AI provides suggestions → You make decisions
  • AI explains possibilities → You choose the best approach
  • AI generates code → You understand and verify it
  • AI finds patterns → You apply domain knowledge
  • Always maintain responsibility for the final product

The golden rule: Never submit code you don’t understand!
